# Onboarding Copy Playbook

## Purpose
Standardized copy patterns for welcome flows, tooltips, empty states, and first-time user experiences.

---

## Onboarding Flow Structure

### Stages
| Stage | Goal | Tone |
|-------|------|------|
| Welcome | Build excitement | Warm, celebratory |
| Setup | Guide through config | Clear, supportive |
| First Action | Drive core action | Encouraging, focused |
| Success | Celebrate completion | Rewarding |

## Tooltip Copy

### Structure
```
Title (what it is)
Description (why it matters)
[Got It] or [Learn More]
```

### Examples
```
Profit Calculator
Instantly see your margins before importing.
[Got It]

AI Suggestions
Products our AI thinks will sell well in your niche.
[Tell Me More]

Bulk Import
Import up to 100 products at once.
[Got It]
```

### Rules
- Max 2 sentences
- Focus on benefit, not feature
- One per feature, don't overwhelm
- Dismissible (one-time or session)

## Coach Marks (Feature Tours)

### Sequence Pattern
```
1/4: This is your Dashboard
See your sales, orders, and performance at a glance.
[Next]

2/4: Find Products Here
Browse thousands of winning products to sell.
[Next]

3/4: AI Suggestions
Our AI recommends products based on your niche.
[Next]

4/4: You're Ready!
Start adding products to your store.

```

### Rules
- 3-5 steps maximum
- Point to actual UI elements
- Allow skip at any point
- Only show once (unless requested)

```
// If has sales
"You made ${revenue} this week. Keep it up!"
```

---

## Constraints

### DO
- Be brief (max 2 sentences)
- Focus on benefit to user
- Celebrate progress genuinely
- Provide escape routes (skip, later)
- Match tone to moment

### DON'T
- Overwhelm with multiple pointers
- Use jargon in early onboarding
- Make skipping feel like failure
- Repeat the same message
- Block core functionality for onboarding
